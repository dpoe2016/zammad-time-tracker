### Step 2: Create Your First Sprint
1. Click **"+ New Sprint"** button
2. Fill in:
   - **Name**: "Sprint 1" (or current week/date)
   - **Start Date**: Today or Monday
   - **End Date**: 1-2 weeks from start
   - **Goal**: "Complete high-priority bug fixes" (optional)
3. Click **"Save Sprint"**

### Step 3: Add Tickets to Sprint
1. Select your newly created sprint from the dropdown
2. **Drag tickets** from the left **Backlog** column
3. **Drop them** in the right **Sprint** column
4. Tickets are now assigned to your sprint!

### Step 4: Set Time Estimates
1. Click **"Edit"** button on any ticket in the sprint
2. Enter estimated hours (e.g., 2.5 for 2.5 hours)
3. Click **"Save"**
4. Sprint stats will update automatically

### Step 5: Start the Sprint
1. Make sure you've added all desired tickets
2. Click **"Start Sprint"** button
3. Confirm the action
4. Your sprint is now active! ğŸ‰

## ğŸ’¡ Quick Tips

### Organizing Your Backlog
- Use the search box to find specific tickets
- Filter by state (New, Open, In Progress, Waiting)
- Higher priority tickets (P3, P4) show with red/orange badges

### During the Sprint
- Continue using regular time tracking on sprint tickets
- Check sprint progress in the stats bar
- Move tickets back to backlog if needed (just drag them!)

### Completing a Sprint
1. When sprint period ends, click **"Complete Sprint"**
2. Unfinished tickets automatically return to backlog
3. Review the final statistics
4. Create your next sprint!

## ğŸ†˜ Troubleshooting

**No tickets in backlog?**
- Go back to main dashboard first
- Wait for tickets to load
- Then open Sprint Planning

**Can't drag tickets?**
- Make sure sprint is selected (not "Backlog")
- Try refreshing the page
- Check if JavaScript is enabled

**Stats not updating?**
- Click the Refresh button
- Check that tickets are in correct state in Zammad
- Verify estimates are saved
